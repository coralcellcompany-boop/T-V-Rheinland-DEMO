# TÜV Rheinland Arabia — Inspection Management System

Multi-tenant inspection management webapp for Saudi Aramco contractors. Implements Third Party Inspection (TPI), Blue Sticker, and (later) Operator Assessment workflows per SRS-IMS-2026-001.

**Stack:** .NET 10 / ASP.NET Core / EF Core / SQL Server / ASP.NET Identity + JWT — Angular 20 (standalone components, signals) + PrimeNG (Aura theme).

**Status:** Phases A → F delivered. **All three SRS services live**: Third Party Inspection, Blue Sticker, and Operator Assessment. Includes auth, clients, equipment with 56 TPI types seeded, certificates with full state-machine workflow, approvals, dashboard, audit log, outbox-driven emails, users admin, inline checklist editor, QuestPDF certificate generation, Blue Sticker stock + auto-issue on Approved + QR + public verification page, **operator candidates registry + assessments with 5-state lifecycle + auto-issued competency cards (TUVR-YY-NNNNNN) + public card verification page (PII-masked)**.

**Architecture:** Clean Architecture (Domain / Application / Infrastructure / Contracts / Api). Custom CQRS, [Stateless](https://github.com/dotnet-state-machine/stateless) for the certificate state machine, hash-chained `AuditLog` via EF interceptor, multi-tenancy via `ITenantContext` + global query filter, NSwag-generated Angular API clients.

## Prerequisites

- .NET SDK 10.x
- Node.js 20+ and Angular CLI
- Docker Desktop (for SQL Server + MailHog)

## Getting started (local dev)

```bash
# 1. Bring up SQL Server (and MailHog)
docker compose up -d
# Wait until 'tuv-mssql' shows (healthy):
docker compose ps

# 2. Run the API (serves Swagger UI at http://localhost:5000/swagger)
#    The first run automatically applies migrations and seeds roles + an admin user.
dotnet run --project src/TuvInspection.Api --urls=http://localhost:5000

# 3. Run the Angular SPA (default http://localhost:4200; pass --port to override)
cd web && npm install && npm start
```

**Reset everything (drops the DB volume and re-seeds an admin):**

```bash
docker compose down -v
docker compose up -d
dotnet run --project src/TuvInspection.Api --urls=http://localhost:5000
# Watch the console for the new auto-generated admin password.
```

**First-run admin credentials.** On the first start the API logs:

```
[WRN] First-run admin credentials:
[WRN]   Email:    admin@tuv-arabia.local
[WRN]   Password: <auto-generated>
[WRN] CHANGE THIS PASSWORD IMMEDIATELY VIA THE UI.
```

Copy the password from the console, log in, and change it. It is logged only once — there is no recovery path other than recreating the user.

**MailHog UI:** <http://localhost:8025> (captures every email the outbox sends).

## Features delivered

| Area | What works |
|---|---|
| Auth | JWT access + refresh, role-based guards (Manager / Coordinator / Inspector / TechReviewer / ClientUser), password lockout, idempotent admin seed |
| Clients | List with search and pagination, create / edit drawer, contract-status pill, allowed-services flags, soft-delete safety |
| Equipment | List with client / Aramco-category / status filters, drawer edit with type & SWL, Excel bulk import (EPPlus), 56 TPI equipment types seeded from SRS §8.1 |
| Certificates | Auto-numbering `IS-NNNNNN-YY-NNNN`, create dialog, state-machine driven workflow with role-gated transitions (Stateless), audit chain via SaveChangesInterceptor, lifecycle timeline UI |
| Approvals | 3-bucket queue (Pending / Rejected / Mine) with live counts |
| Dashboard | Live KPIs (8 cards), recent-activity feed from the audit log, hero with personalized greeting |
| Audit | Hash-chained `AuditLog` in a separate DbContext, surfaced through `/api/dashboard/activity` |
| Outbox | Background-friendly outbox table; emails enqueued on `ClientSent` |
| Outbox processor | Polls every 5s, dispatches to typed handlers, exponential backoff, capped at 8 attempts |
| Users admin | Manager-only `/admin` page — create/edit users, assign roles + clients, reset password |
| Checklist editor | In-place table editor on the cert detail page (only when state is Draft/Rejected); JSON-backed |
| PDF certificate | QuestPDF Annex-1 renderer at `/api/certificates/{id}/pdf` |
| Blue Stickers | Stock register, procure-batch, void; auto-issued (`TUVR######`) when an Aramco-categorized certificate hits Approved |
| QR codes | `/api/public/qr/{stickerNo}.png` — public, cached, encodes the verification URL |
| Public sticker verification | `/verify/{stickerNo}` — mobile-first, anonymous, mask-equipment-ID, no PII |
| Operator Assessment | Candidates registry, assessments with Stateless state machine (Draft → Submitted → Approved/Rejected), score editor, transition timeline |
| Competency cards | Auto-issued (`TUVR-YY-NNNNNN`) on Assessment.Approve when result=Pass; QR PNG + public `/verify-card/{cardNo}` page with PII masking (name → "Mohammed A.", ID → ••••••6789) |
| OpenAPI | NSwag spec at `/swagger/v1/swagger.json`, UI at `/swagger` |
